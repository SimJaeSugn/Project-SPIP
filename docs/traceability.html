전송: Electron IPC (과도기 HTTP 보존). 제품이 로컬 웹앱 → Electron 데스크톱 앱으로 이전 완료되어 전송 계층이 HTTP → Electron IPC로 바뀌었다. 사용자 표면(R-04 루트 설정·R-12 열기·R-15 진행·R-16 재스캔)의 구현 위치·검증을 Electron 기준으로 갱신했다(아래 해당 행에 표기). 도메인 로직(walker·detector·collectors·scanController·pathGuard·safeExec)은 전송 무관하게 그대로 재사용된다. IPC 계약 정본: docs/architecture/electron-migration.html §4. 과도기 HTTP 서버(server.js·lib/server/* · npm run start:web)는 아직 보존되며 후속 제거 예정(코드리뷰 P2-2). 요구 상태는 전부 구현 완료 유지.
막바지 버그픽스 2건 반영(상태 변동 없음, 검증 칸 갱신). ① 스캔 후 무한 로딩 — scanController.start()가 finalizing/done/error 터미널 phase를 push하지 않아 스캔 완료 후 대시보드 전환이 안 되던 문제(R-15 진행 push·R-16 재스캔 후 갱신 관련) → 터미널 phase emit 추가(세대 가드 유지)·회귀 테스트 test/scanController-onprogress.test.js. ② 패키징 앱 실행 크래시 — build/afterPack.js의 EnableEmbeddedAsarIntegrityValidation fuse가 Windows+electron-builder 24.x에서 FindResource FATAL을 유발(패키징/배포 관련) → 그 fuse만 비활성(나머지 보안 fuse 유지)·app:// 핸들러 fs 직접 읽기(asar-aware)·did-fail-load 시 창 강제 표시. 실 GUI에서 창 정상 표시·동작 확인됨. 버전 package.json 1.0.1.
