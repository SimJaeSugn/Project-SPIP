2. 차트 vs 메일 완료부 비대칭 (흐름)

두 폴링이 무변경 처리에서 다른 이유 — 정당한 차이임을 확인.

결론: 메일은 fetch 중 busyMail로 보류가 쌓이므로 완료부에서 flush가 필요하고, 커밋은 fetch 전 deferred로 게이트되어 별도 pending 추적으로 따라가므로 완료부 flush가 불필요하다. 비대칭은 의도적이며 정확하다(개선 #1로 문서화만 권고).

5. 테스트

M11 신규 12건 + patchRegion 기존(bypassDefer 반영) 갱신 포함 36건 전부 통과. 순수 함수(mailSummaryKey·patchRegionPlan)는 단위 동형 검증, DOM 배선은 소스 계약(정규식)으로 강제 — jsdom 0-의존 정책 유지. 전체 스위트 748건 통과 보고와 정합.
