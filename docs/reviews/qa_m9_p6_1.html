R-31 커밋 차트 5분 폴링은 동작·통합·회귀 측면에서 결함 없이 수용 기준을 충족한다. 최우선 수용 기준인 "홈 비활성 시 git 호출 0"은 두 겹의 방어로 보장된다 — (1) 순수 shouldPollCommit(view, visible)가 view==='home' && visible!==false일 때만 true이고, 뷰 전환(render 말미 _lastView 변경 시)과 visibilitychange 양쪽에서 syncHomePolling()이 호출돼 게이트가 false면 clearInterval로 타이머를 실제 정지한다(setInterval 자체가 사라져 git 콜백이 더는 발화하지 않음). (2) 타이머가 살아있더라도 폴링 콜백 maybeAutoRefreshCommit 첫 줄의 store.state.view !== 'home' 이중 가드가 git 조회를 차단한다. 폴링 주기 상수 COMMIT_POLL_MS = 300000(5분)은 메일 60000(1분)보다 길다. 폴링 콜백은 RG.deferred()(조합·드래그·오버레이·busy) 중 1회 건너뛰고, 완료 반영은 refreshCommitActivity → RG.coalesce.release() 종착이라 R-25/R-26과 정합한다. 메일 60초 폴링은 별도 타이머(mailRefreshTimer)로 독립이라 회귀 0. npm test 660건 전부 통과(신규 R-31 7건 포함), 0 fail. node --check 구문 OK.
