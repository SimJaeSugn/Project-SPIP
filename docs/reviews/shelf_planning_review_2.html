1. 재검증 상세

P1 — SCHEMA_VERSION 이행 [PASS]

로드맵 SH-1 uiStateStore.js 행(line 494)이 빌드 deliverable로 승격됐고, 1회차 지적을 정확히 포섭한다: ① 입력 obj.schemaVersion 판독 ② schemaVersion < 2 또는 부재(undefined)(=레거시)면 ['shelf','shelfWide']를 hiddenWidgets에 union(멱등) ③ 출력 schemaVersion=2. 사용자가 이미 unhide한 v2 파일은 재숨김 안 함(의도 보존). SCHEMA_VERSION 1→2 명시. 핵심 함정인 undefined < 2 === false를 본문에서 명시 경고하고 "undefined를 명시적으로 레거시로 분기"로 못박았다 — 코디네이터 확인점 충족. 테스트(line 500)도 (a)구버전 union (b)schemaVersion 부재(undefined) 레거시 (c)v2 unhide 재숨김 안 함 (d)v2 무변경 4케이스를 전수 명시. 실측 코드(normalizeState가 입력 schemaVersion을 안 읽음)와의 간극이 빌드 단계에서 정확히 해소됐다.

P2 — og:image 100KB 통일 [PASS]

설계 SSOT 전수 grep 결과 96KB 잔존 0. §2 품질카드·§9 크롤 SVG·§11 본문·§13 응답한도·ADR-SH-2 결과· §16 리스크표 모두 100KB로 통일됐고 §16 Q1은 "[해소] 100KB·SVG 제외 확정(PM #4)"로 마감. 로드맵 측도 100KB 일관. 코디네이터 확인점(96KB 잔존 없음) 충족.

P2' — SSOT file 열기 정합 [대부분 PASS · 잔여 2곳]

정정 확인: §2 다이어그램(line 232 "파일→OS 기본앱"), §8 IPC표(line 470 "file=shell.openPath(OS 기본앱 — PM #2 확정)"), §16 Q4(line 773 "[해소] 파일=OS 기본앱(shell.openPath) 항상")가 PM #2로 정정됐다.

잔여(비차단). SSOT의 비운영 prose 2곳이 미정정이다:
  • §10.3 line 591: "folder/file → safeExec(resolveTool('code'),[real],...) ... 도구 미설정 시 shell.openPath 폴백" — file까지 VS Code 우선으로 기술(PM #2와 모순).
  • §13 H-2 line 635: "folder/file 열기는 safeExec 절대경로·shell:false..." — file이 여전히 safeExec 기반으로 기술(PM #2와 모순).
이 두 줄은 "열기 정책" 상세·보안 체크리스트라 security-reviewer/구현자가 직접 참조할 가능성이 있어 정리가 바람직하다. 다만 운영 구현 지침(로드맵 SH-2 line 519: "file → shell.openPath(real)(OS 기본앱)" + IPC 계약 §셸프)이 명확·정확해 빌드는 올바르게 진행 가능하다 → 비차단 잔여로 분류.

P3 — validateHttpUrl 회귀 [PASS]

로드맵 SH-1이 (line 496) 반환형 {ok,value,external} 불변 + validateBriefingUrl 별칭/래퍼 무변경을 명시하고, 테스트 deliverable로 (line 501) validateHttpUrl 단위(scheme/임베디드 자격/길이/maxLen 경계·반환형) + (line 502) 기존 test/ai/config-briefing.test.js 그린 유지를 못박았다. 회귀 안전망이 계획에 명시됨 — 통과.

경미 — 표기 정리 [PASS]

CSP 출처가 설계 머리말(line 92)에서 "public/index.html(CSP meta — img-src data:)"로 정정됨. 스케줄러 기본값 §12(line 624)·§16 Q2(line 771)가 "기본 ON + 끄기 토글(PM #1)"로 해소. og:image 96KB 불일치 정합 완료.

2. 잔여 반영 항목 (비차단)

우선무엇위치문제 / 권고
권장(비차단) SSOT file 열기 prose 정정 설계 §10.3 line 591 · §13 H-2 line 635 문제: 두 줄이 file 열기를 여전히 VS Code/safeExec(code) 기반으로 기술 — PM #2(file=OS 기본앱 shell.openPath 항상) 및 이미 정정된 §2/§8/§16과 모순. 권고: §10.3은 "folder → safeExec(code) / file → shell.openPath(OS 기본앱, PM #2)"로 분기 기술, §13 H-2는 "folder 열기는 safeExec... / file은 shell.openPath"로 분리. 운영 빌드 지침은 이미 정확하므로 구현 착수를 막지 않으나, SSOT 일관성·SH-3 보안 게이트 입력 정확도를 위해 함께 정리 권장.
