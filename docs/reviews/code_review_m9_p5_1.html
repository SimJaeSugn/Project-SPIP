3. :has() 셀렉터 견고성 평가

코드베이스 첫 :has() 사용(grep로 다른 사용처 없음 확인). Electron 번들 Chromium은 105+ 이상이라 :has() 지원이 보장되는 단일 런타임 — 미지원 우려는 사실상 없고, 사용 자체가 적절하다.

  • 스코프 안전. 두 규칙 모두 .settings-2pane를 가진 모달에만 매칭(css:715~716). 메일 본문 모달·업데이트 모달 등 다른 buildModal 산출물은 영향 없음 — 오염 없음.
  • 폴백 기능 안전. 가령 :has() 미지원이라 가정해도 .settings-2pane{display:flex} 골격(css:717)·nav·pane 스타일은 클래스 셀렉터라 그대로 적용 → 2-pane 레이아웃·스크롤 정상. 잃는 것은 "설정 모달만 820px·body 패딩 제거" 두 가지 시각 보정뿐(기능 무손상). progressive enhancement로 올바르게 설계됨.
  • 특이도 함정(개선 #2). 같은 효과를 .modal--wide-settings 같은 명시 클래스로도 낼 수 있는데 :has()를 택해 wide:true가 무의미해졌다. 의도는 맞으나 두 너비 규칙의 승패가 특이도에 암묵 의존 — 주석으로 명시 권고.
