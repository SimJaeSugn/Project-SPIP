1. 요약

R-33 SVG 커밋 차트는 스캔 유래(신뢰 불가) 데이터를 라벨·툴팁·접근성 텍스트로 표시하지만, 값은 예외 없이 textContent로만 들어가고 수치는 commitChartModel에서 Number()+Number.isFinite로 sanitize되어 좌표/치수 속성에 흐른다. 색·치수 속성은 데이터가 아닌 고정 팔레트/모델 상수에서 setAttribute되며, 외부 리소스(xlink:href·이미지·CDN·폰트) 참조는 0이다. 이벤트는 전부 addEventListener(인라인 on* 0, eval/Function 0)다. 설계 리뷰 M-2의 권고 ①~⑥이 모두 구현에 반영되었다.

발견 건수: 치명 0 · 높음 0 · 중간 0 · 낮음 0 · 정보 2(관찰 사항, 차단 아님)

2. 신뢰 경계 · 데이터 흐름

붉은 점선 = 신뢰 경계. 스캔 유래 데이터는 commitChartModel에서 정규화·클램프된 뒤에만 렌더로 흐르고, 텍스트는 항상 textContent로 격리된다. CSP가 잔여 위험에 대한 2차 방어막.

4. 점검 상세 (M-2 중심)

① 텍스트 인젝션 — textContent 전용 (L-1)

  • SVG <title>(스크린리더), 외부 툴팁 div, 하단 요일 라벨 span — 세 경로 모두 textContent로만 설정. 커밋 일자 label과 수치 count가 마크업으로 파싱되지 않는다.
  • 차트 빌더 본문에 innerHTML/insertAdjacentHTML/outerHTML 사용 0건 (정적 검사·grep 확인). 라벨 span은 el({text}) 헬퍼를 경유하며 헬퍼도 node.textContent만 쓴다.
  • SVG 텍스트 노드 내부에 <a>/<animate onbegin> 같은 SVG 내 스크립트 벡터가 끼어들 표면 없음(문자열이 노드 텍스트로만 들어감).

② 속성 인젝션 — 고정 팔레트/sanitize 수치만

  • fill은 데이터가 아닌 CHART_PALETTE(bar/barLast/barHover/stub) 상수에서만 결정. 색을 데이터-주도로 받지 않으므로 속성 인젝션 표면 없음.
  • x/y/width/height는 commitChartModel이 산출한 수치를 String(b.x) 형태로만 주입. count는 Number() 변환 후 Number.isFinite && n>=0 게이트로 NaN/Infinity/음수/비수치를 0으로 클램프 → 속성에 NaN·스크립트 유입 0. (테스트로 검증됨)
  • 동적 aria-label은 호출부 고정 문자열('최근 7일 커밋 빈도')이며 데이터 미경유.

③ 외부 리소스 0

  • 차트 SVG에 xlink:href·<image>·외부 url(...)·웹폰트 참조 0건. https?:// 리터럴 0건(정적 검사 확인).
  • styles.css 차트 규칙(.commit-chart*)도 외부 url()·@import 없음 — 색·그림자 전부 리터럴/var(--accent).
  • CSP default-src 'none'; connect-src 'none'; img-src 'self' data:와 정합. 자작(self-host 라이브러리조차) 확인 — 외부 차트 라이브러리 의존 0.

④ 이벤트 핸들러 · 동적 코드

  • 호버/포커스(mouseenter/mouseleave/focus/blur) 전부 addEventListener 경유(내부 on() 래퍼가 핸들러를 추적). 인라인 on* 속성 0. CSP script-src 'self'가 잔여 인라인 핸들러를 2차 차단.
  • 데이터 기반 eval/new Function/setTimeout(string) 0건.
  • destroy()가 모든 핸들러를 removeEventListener로 정리(RG.widget destroy/recreate) — 누수·중복 바인딩 없음(가용성 측면 양호).
