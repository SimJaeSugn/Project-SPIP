1. 회귀 0 — PASS

  • RG 공개 API 시그니처 불변: RG 모듈 반환은 { composition, deferred, preserve, widget, coalesce }(5841) 그대로. preserve는 capture/restore에 patchRegion이 추가됐을 뿐 기존 메서드 시그니처 불변(5725). widget은 define/destroyAll/mountAll 불변, _destroyById/_mountById는 밑줄 내부 메서드로 신설(5784·5793). coalesce request/flushIfPending/release/cancel 불변(5834).
  • render() 경로 불변: render의 destroyAll→capture→replaceChildren→restore→mountAll 골격 미변경. 뷰전환·재스캔·applyProjectsUpdate·오버레이·메일폴링·설정탭 어느 경로도 patchRegion으로 전환되지 않음 — 차트 폴링 완료부(2277)만 옵트인 전환.
  • 최초/수동 로드 진입 render 유지: refreshCommitActivity는 !opts.silent && view==='home'일 때만 진입 render(2261). 최초 로드(2243)·수동 호출은 silent 미지정이라 로딩 render 유지 — 기존 동작 보존.
  • renderGuard.js 변경 0: git status·git diff -- lib/common/renderGuard.js 모두 비어 있음. M10 작업 트리 변경은 app.js·styles.css·test 3종에 국한. 순수 정본 불변 확인.
