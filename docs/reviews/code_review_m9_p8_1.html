사소 라벨/수치 문자열 조립 중복 — title((label||'·')+': '+count+' 커밋')과 tip((label||'·')+' · '+count+' 커밋')이 거의 동일 패턴 2회. app.js:1311,1316 구분자만 다른 동일 의미 텍스트. 작은 헬퍼(barText(b, sep))로 묶으면 표기 변경 시 단일 출처. 신호 대비 가치는 낮으니 선택.
사소 createElementNS 반복 — chartBars에서 svg/rect/title을 raw createElementNS+setAttribute 나열. 같은 파일에 svg 노드 헬퍼(svg())가 있으나 viewBox 24 고정이라 재사용 불가. app.js:1281–1312 설계가 "svg()는 차트에 부적합"이라 새 빌더를 둔 판단은 타당. 다만 setAttribute 다발은 작은 로컬 헬퍼(ns(tag, attrs))로 묶으면 가독성↑. 현 코드도 충분히 읽히므로 필수 아님.
사소 colW 음수 미가드 — colW=(viewW-gap*(n-1))/n. 극단적으로 작은 width 또는 큰 gap opts에선 음수 가능. app.js:695 실제 호출은 기본값(240/gap6/n7→29.1)·테스트(200)만이라 도달 불가에 가깝고, SVG는 음수 width를 무시할 뿐 크래시는 없음. 방어적으로 Math.max(1, ...) 정도면 충분. 우선순위 낮음.

3. 가독성 · CSS · 단순성

  • 순수 모델/빌더 분리 적절 — commitChartModel(좌표·치수·sanitize, 순수)과 chartBars(DOM·핸들러)가 깔끔히 분리돼 헤드리스 테스트 가능. 설계 의도와 일치.
  • 죽은 코드 없음 — 기존 div 인라인 height 막대는 호스트 .commit-chart-host로 완전 대체됐고 잔재 없음(grep 확인). 차트 클래스는 모두 새 SVG 경로에서만 사용.
  • CSS 일관 — .commit-chart__* BEM 네이밍, var(--accent) 포커스, 다크 툴팁(#1c1917) 모두 주변 스타일과 일관. transition: fill .12s로 호버 강조가 부드럽다.
  • 다크톤 팔레트 — CHART_PALETTE는 라이트 카드(홈은 밝은 카드 배경) 위 인디고 계열로, 홈 생산성 카드 톤과 맞물린다(차트는 밝은 카드 내부 위젯이라 다크 보고서 톤과 별개). 일관성 문제 없음.
  • round1 수치 정리 — 좌표를 소수1자리로 라운드해 SVG 속성 문자열이 깔끔. 적절한 단순화.
