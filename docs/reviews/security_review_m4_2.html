C-4(중간)·L-1(낮음) 모두 해소 확인. 신규 위협·회귀 0 → 보안 게이트 전면 통과. C-4는 _forceError가 이제 this.running=false로 락을 즉시 강제 해제(scanController.js L128)하여 영구 409 데드락이 제거됐다. 1회차에서 지적한 "버려진 orphaned 스캔의 늦은 settle 오염" 위험은 세대(scanId) 가드(_isActive)로 봉쇄됐다 — finally·진행 병합(_merge)·finalizing(writeSnapshot/store.load)·done 전이가 모두 _isActive(myScanId) 통과 시에만 상태/락을 건드린다. 1회차가 지적한 거짓 안심 테스트는 영원히 resolve 안 되는 실제 hang scan 모의 + 세대 가드 오염 테스트 2건으로 대체됐고(테스트 213건, +12), 직접 프로브로 watchdog 발화 후 running===false·후속 acquire 성공·늦은 orphan settle의 무오염을 실측했다. L-1은 shortenPath가 드라이브 루트 단독(/^[A-Za-z]:$/)을 일반 라벨("드라이브 루트")로 치환해 드라이브 문자 노출을 차단한다. 회귀: size 항상 로드(BUG-M4-01)는 저비용 deps 카운트만 추가하고 비용 큰 트리 측정은 여전히 enabled 게이트 + 예산 가드 뒤에 있어 N-01/H-3 무손상. all-drives 루트 제외(P2-4)는 시스템 루트만 추가 차단(드라이브 루트는 세그먼트 길이상 미해당) — C-2 강화일 뿐 약화 아님.

1. C-4 · L-1 해소 확인 (코드 + 테스트 + 직접 프로브)

항목1회차수정 내용(코드 위치)실측 입증(직접 프로브)2회차
C-4-a
락 해제		중간 _forceError가 this.running=false를 함께 해제(scanController.js:128). watchdog 콜백이 _isActive(myScanId) 확인 후 _forceError 호출(L149-153). scan settle의 finally를 더 이상 기다리지 않음. 프로브 [1]: 영원히 resolve 안 되는 scan 주입(watchdog 40ms) → watchdog 발화 후 running=false, phase=error 측정. finally는 절대 도달 못 하는데도 락 해제됨 → 영구 409 데드락 제거 입증. 해소
C-4-b
세대 가드		(신규권고) _isActive(scanId)=running&&state.scanId===scanId(L102-104). start가 myScanId 캡처(L143), finally(L196)·_merge(L113)·finalizing/done(L172,183)·catch(L186)가 모두 자기 세대일 때만 상태/락 변경. 버려진 run은 writeSnapshot/store.load도 미실행(L172 early return). 프로브 [2][3]: watchdog 강제해제 후 후속 acquire 즉시 성공(새 scanId)·정상 done. 이후 버려진 1회차 scan을 늦게 settle → phase=done 불변, scanId 불변, store.load 호출 1회 유지(orphan 미실행), counts=후속값(orphan 999 미반영). 오염 0. 해소
C-4-c
회귀 테스트		거짓안심 1회차 거짓 안심(작은 트리 빨리 끝나 finally가 락 해제) 제거. 신규 2건: m4-rescan.test.js:264(영영 hang하는 scan → watchdog 즉시 해제 단언), :293(세대 가드 — 늦은 settle 무오염·load 1회 단언). npm test 213/213 pass(1회차 201 → +12). 신규 테스트가 실제 hang을 모의해 watchdog의 락 해제를 직접 단언(finally 의존 제거). 해소
L-1
드라이브루트		낮음 shortenPath가 출력 1세그먼트가 /^[A-Za-z]:$/면 '드라이브 루트' 반환(scanController.js:47-49). 프로브 [4]: shortenPath('C:\\')→"드라이브 루트", 'D:\\'→"드라이브 루트". 일반 경로(...\Windows\System32→"Windows/System32") 정상. 테스트 :93-94 단언. 드라이브 문자 노출 0. 해소
