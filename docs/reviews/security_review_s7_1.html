3. 이월 항목 V-1·V-2 — 실측 판정

V-1. realpath·절대경로의 TOCTOU 잔여 — 해소

요구 검증~spawn 사이 심링크 교체 창 최소화 + 심링크/junction/UNC/대소문자/NFD/확장경로 거부 회귀
증거 actionHandlers.open은 ① canonicalize(project.path)로 실경로 real 산출(소멸 시 410 PATH_GONE) → ② isAllowed 화이트리스트 정확 일치 검증 → ③ 검증 직후 같은 real 값을 safeExec(codeBin, [real]) 인자로 전달. 원시 project.path를 재해소하지 않고 검증 산출물을 그대로 실행해 verify→spawn 재해석 창을 제거(설계 "표면 축소"를 코드로 이행). walker도 자식 realpath 해소 후 visited 집합 비교.
실측 pathGuard.test 6건 통과(소멸→null, ../차단, 접두사함정 차단, 대소문자 폴드, 심링크 실경로 해소, NFD). 프로브: UNC/드라이브상대/존재않는 확장경로 모두 null로 거부, 실존 \\?\C:\Windows 정상 해소. 크래시 0.
잔여 단일 사용자·로컬 루프백·메타데이터 only 모델에서 동일 디렉터리 타 사용자 격리는 RC-1로 수용된 비목표(설계 명시). 본 모델 한도 내 TOCTOU 실위험 무시 가능.

V-2. 적대적 입력 가드 + cwd 위장 바이너리 — 해소

요구 거대/과중첩 JSON skip·ReDoS 0·제어문자 정제·위장 git.exe/.bat 미실행(pwned 미생성)·.bat 거부 실측
실측(회귀) 2MB package.json → TOO_LARGE skip · 깊이 200 JSON → TOO_DEEP 거부 · <img onerror>+제어문자+5000자 name → 정제(제어문자 제거·절단) · 위장 git.bat(@echo EVIL > pwned.txt) 둔 폴더를 git collect → pwned.txt 미생성(절대경로 git + -C로 위장 미선택) · .exe 절대경로만 허용, .bat 거부 · resolveBin cwd 위장 미선택.
실측(프로브) parseStatus에 200,000자 브랜치명 입력 → 0ms 완주(선형, ReDoS 없음, ahead/behind 정상). clampString 제어문자(\n\t) 공백 치환 확인.
판정 모든 가드가 파싱/격리 앞단에 실재하고 동작 실측됨.
