4. 후속 추적 (PASS 조건 아님 — 비블로커)

  • P3-1 실행 추적: 본 PASS와 동시에 docs/api-contract.md를 §8.1~§8.4대로 갱신해야 단일 출처 표류가 닫힌다(현재 rescan=404·totalBytes:null·size always skipped로 남아 있음). 설계 자체의 결함은 아니며 절차(§8.5)에 명시됨 — PM이 갱신 실행을 누락하지 않도록 추적.
  • 구현 시 검증 항목(설계 블로커 아님): (1) store.load() 원자성·idempotent 실제 동작, (2) walker 전역 maxDirs/timeBudgetMs + ABS_MAX_DEPTH=64 + all-drives 조합의 N-01 예산 준수 — §6.1이 벤치(V-2 연장)로 실증하라고 명시했으므로 qa-engineer 실측 단계에서 확인.
  • 보안 최종 판정: M4-H-1/H-2/M-3, checkReadAccess, currentPath 축약의 위협 모델 적정성은 security-reviewer 2회차 재검 소관. 본 보고는 추적성(통제가 설계에서 약화 없이 일관 언급되는가)만 점검했고, 통제 언급은 일관 유지됨.

5. 확인 필요 (PM 판단)

  • 설계 미해결 Q2~Q5(size 기본 on/off·폴링 주기·depthLimit 기본값·a11y 검증 도구)는 PM 결정과 충돌 없이 권고+확인 요청으로 정리됨 — 별도 상신 불필요. Q1은 PM 확정으로 종결됨.
  • 본 검증으로 상신할 신규 블로커 쟁점 없음.

6. 검증 범위 / 경계
