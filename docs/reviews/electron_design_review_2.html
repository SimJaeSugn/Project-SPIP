P1 해소 검증 — F-1 / F-9 (scanController onProgress) 해소

1회차 P1의 핵심: start(opts)가 onProgress를 인자로 안 받고 167행에서 _merge를 하드코딩하므로 "무수정 중계" 불가. 실코드와 v1.1을 한 줄씩 대조했다.

검증 포인트실코드 사실v1.1 반영결과
167행 정합scanController.js:167 = onProgress: (p) => this._merge(p, myScanId) (정확히 일치)§4.3 의사코드(변경 전/후) 및 §3 표가 이 라인을 정확 인용OK
"최소 수정" 재분류start 시그니처에 onProgress 없음 → "무수정" 거짓§3 표 scanController 행 = 최소 수정 (onProgress 1개 추가)로 재분류. 헤더 요약·R-15 행도 일관OK
_merge 보존167행 _merge가 내부 state 병합 담당, 세대 가드(_isActive) 적용변경 후 의사코드: this._merge(p, myScanId) 먼저 무변경 호출 후 opts.onProgress 추가 호출 — 내부 동작 보존OK

F-9: 사적 API 직접호출 제거_merge는 _ 접두 사적 메서드main 의사코드(§4.3 line 340)는 onProgress: (snap)=>win.webContents.send('spip:scanProgress', snap) — 공개 콜백만. _merge 직접호출 없음OK

테스트 동반—§3·§10.2·단계 5에 "onProgress 파라미터 단위테스트 + _merge 보존 검증" 신규 추가 명시OK
소결: P1 해소. 의사코드가 실제 167행 구조(_merge + 세대 가드 + shortenPath)와 정합하며, "최소 수정 1점"으로 정직하게 재분류됐다. main이 사적 _merge를 부르지 않아 캡슐화도 보존(F-9). 단계 5 착수 차단 해제.

회귀 검증 — 드롭/추가 변경이 요구·shape에 회귀를 만들지 않는가 회귀 없음

변경회귀 점검결과
browseDir 드롭R-04(루트 지정)는 pickFolders(네이티브 dialog)+addRoots(직접입력)+removeRoot 두 경로로 충족 유지. browseDir는 신규 채널이라 기존 요구 충족 경로에 부재 → 드롭이 R-04 회귀 아님. §4.1/§5/§11 일관회귀 없음
onProgress 추가R-15(진행 표시)는 push로 충족. start에 선택적 파라미터 추가 — 기존 _merge·watchdog·세대 가드·in-flight 락 무변경(R-16 정합). ScanProgress shape는 status() 그대로 push되어 불변(phase/scanId/dirs/found/currentPath/elapsedMs/startedAt/counts/note). §4.1 푸시 페이로드와 실코드 status() 반환 필드 일치회귀 없음
CSP 강화§6.1은 HTTP 헤더 CSP만 제거하고 §6.2에서 메타+세션헤더로 "전 디렉티브 보존+강화"(축약 금지) 명시 → 기존 staticHandler CSP 대비 약화 아님. connect-src 'none'는 IPC 전용이라 정합. R-10/R-11(렌더·필터)은 script-src 'self'로 영향 없음(인라인 스크립트 미사용 전제)회귀 없음
Project shape§4.1 노트: "Project 항목 shape 불변(api-contract §Project 그대로)". 스냅샷은 도메인 산출물, 전송 무관 — getProjects 응답 {schemaVersion,generatedAt,hasSnapshot,projects[]} 유지불변
R-05/R-09/R-12rescan({allDrives})→allowAllDrives 게이트, rescan({withSize})→size collector, open(id)→getById→pathGuard→safeExec — 전부 재사용 도메인 호출 유지. §11 추적표·§4.2 검증체인이 actionHandlers 4~6단계와 정합회귀 없음
소결: 드롭·추가·강화 모두 기존 요구 충족 경로와 도메인 shape에 회귀를 만들지 않는다. 변경은 전송층에 국한되고 도메인은 onProgress 1점 외 무수정.
