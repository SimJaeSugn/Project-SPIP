요약 — 프런트 applyHomeLayout와 백엔드 normalizeHomeLayout은 동일 규칙·동일 출력(화이트리스트 필터 → 중복 제거 → 누락 보충, 항상 7-순열)으로 동형 확인. IPC 인자 shape({ids})과 응답({ok,homeLayout}) 소비가 정확하고, getUiState.homeLayout 적재도 정확. R-25 보류 게이트(_dragging)가 폴링·라이브 push 양쪽 렌더 경로를 모두 막고, onEnd 마이크로태스크(R4)로 destroyAll 충돌을 회피한다. F-4 정량 기준은 CSS(column-count·break-inside:avoid·reduced-motion)·노드 이동 방식으로 정적 충족. 자동 검증 가능 범위는 전부 PASS이며, 실제 포인터 드래그·재진입 영속은 헤드리스 환경 한계로 수동 스모크에 위임한다.
