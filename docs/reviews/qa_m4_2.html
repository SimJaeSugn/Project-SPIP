6. 경계면 교차 — size shape ↔ 프론트 "미측정"/실값 분기 — 일치

실서버 기본 스캔 출력 project(alpha, status:'skipped'+deps:2/devDeps:1+totalBytes:null)를 프론트 toViewModel에 직접 투입해 분기를 실측.

vm.sizeStatus = "skipped"   vm.deps = 2   vm.devDeps = 1   vm.totalBytes = null
sizeStatusLabel(skipped,null) = "미측정"           // 용량은 미측정 라벨
depsHas (typeof deps==='number'||...) = true       // 의존성 행 렌더 → "2 deps · 1 devDeps"
핵심 정합: 프론트 depsLabel(app.js:1396)은 typeof vm.deps==='number'로 판정하며 sizeStatus와 독립. 1회차에는 deps가 null이라 "미측정"으로 떨어졌으나, 이제 실값이 들어와 의존성 행이 정상 표기된다. 동시에 용량(measured = ok|partial)은 skipped라 "미측정"·kpi__value--na 유지 → 의존성은 표시, 용량은 미측정이라는 계약 의도(api-contract §Project shape "deps/devDeps는 size 비활성이어도 채워질 수 있음")가 경계면 양쪽에서 일치. 프론트 코드 무변경으로 이미 호환.
