PASS (통과)

3.3 ADR 정합

ADR-002(git 절대경로 H-2)·ADR-003(설정 위치 P1-2/M-2)·ADR-004(서버 M-1/L-1/P1-1)·ADR-005(스냅샷 200 P2-5)가 rev.2로 갱신되고, ADR-006(부분신뢰 입력 방어)이 신설됐다. 모든 ADR이 "맥락→대안→결정→근거→결과" 형식을 유지하며, ADR-006은 대안 3안(입력게이트/격리만/샌드박스) 비교·RC-2 인정과의 정합을 갖췄다. 과설계(샌드박스 C)를 명시 배제해 N-04(의존성0) 정신과 충돌하지 않는다. ADR 품질 회귀 없음.

회귀 결론: 보안 반영은 기존 추적성·모듈 경계·ADR 품질을 훼손하지 않고 가산적(additive)으로 적용됐다. P1·P2 해소와 보안 시정이 서로 모순되는 지점(예: P1-1 정적 규약 ↔ §11.6 CSP "동일 출처 /static/*", P1-2 위치 ↔ §11.4 권한)도 상호 정합한다.

4. 잔여 관찰 (블로커 아님 · 참고)

판정에 영향을 주지 않는 경미 사항만 기록한다. 모두 1회차 P3 수준이거나 그 이하로, 구현·후속 단계에서 자연 해소된다.

  • 포트 기본값·언어 가중치: §15 미해결 질문에 잔존(예시 7421). 1회차에서 "블로커 아님" 동의한 항목이며 여전히 타당. 탐지율 ≥95% 벤치(test/fixtures)로 튜닝 대상.
  • %APPDATA% 로밍 소지: ADR-003 결과에 "캐시만 %LOCALAPPDATA%로 옮기는 것은 M4 재검토 여지"로 트레이드오프가 정직하게 기록됨 — 현 규모 무시 가능, 결정 추적성 양호.
  • UserStories 정합: 1회차 P3-1과 동일하게 본 검증 범위 밖. 로드맵/세부계획 검증 단계에서 스프린트 매핑과 함께 확인 권고(현 판정 무영향).
  • 보안 상세 적정성(realpath/TOCTOU·CSP 우회·세션토큰 강도 등): security-reviewer 소관. 설계서가 §11에 명시적으로 그 위임을 표기 — 적절.

5. 결론

PASS

1회차 조건부 PASS의 전환 조건이었던 P1 2건(정적 URL 규약 통일·설정 위치 확정)이 모두 해소됐고, P2 5건도 전부 반영됐다. 보안 검증 1회차 반영(§5·§11 대폭 수정, ADR-006 신설)으로 인한 추적성·정합 회귀는 없으며, §14 MVP 매핑 누락 0건이 유지되고 신규 모듈/계약이 디렉터리 구조·요구ID와 정합한다.

아키텍처 설계서·ADR은 추적성·기획 정합·모듈 경계·ADR 품질·디렉터리 일관성·보안 신뢰경계 모델에서 착수 가능한 완성도에 도달했다. 별도 재검증 없이 개발 로드맵/세부계획 단계로 진행 가능하다. 상세 보안 적정성은 security-reviewer, 빌드/실행 검증은 qa-engineer에게 넘긴다.
