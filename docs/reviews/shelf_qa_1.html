3. 상태별 UX — PASS · LOW 1건

  • empty/few/many 분기: shelfStateFlags가 {count,hasItems,isEmpty} 산출. isEmpty → shelfEmpty(), hasItems → 패널 행. "many"는 shelfSpineW(n)가 6개 초과부터 칸당 3px 축소(최소 42px)로 시각 대응 — 초안 spineW 그대로.
  • loading 플레이스홀더: 최초 list 적재 중(!loaded && busy && 0건)은 스켈레톤 스파인 4개, 컴포저 입력 중(cState==='loading')은 행 끝에 shelfLoadingSpine() 추가 + 스피너 + crawlingLabel.
  • 입력 idle/loading/error: shelfComposerVM이 placeholder·테두리색·상태라벨 분기, error는 role="alert" 슬라이드업 박스.
  • error 매핑 한국어: BLOCKED_HOST/CRAWL_FAILED/PATH_DENIED/PATH_GONE/BAD_INPUT/LIMIT/UNSUPPORTED_TYPE/ NOT_FOUND/OPEN_FAILED/INTERNAL/FORBIDDEN 전부 친화 문구 + type별 default.
BUG-SHELF-01 (LOW, frontend) — CRAWL_PENDING 잔재. SH-3에서 URL 크롤이 구현돼 backend는 CRAWL_PENDING을 더 이상 반환하지 않는다(코드 전수 확인: shelf.js add/refresh는 crawl 결과코드 BLOCKED_HOST/CRAWL_FAILED/BAD_INPUT만 반환). 그런데 다음 3곳에 잔재가 남음:
  • public/app.js:729 — case 'CRAWL_PENDING': return 'URL 즐겨찾기는 곧 지원돼요…' (도달 불가 죽은 코드 + URL이 이미 지원되므로 문구가 사실과 반대).
  • electron/ipc/shelf.js:8-9 — 파일 docstring이 SH-2 시절 "add/refresh에서 CRAWL_PENDING 반환"을 현재 동작인 양 서술(stale 주석).
  • test/shelf-widget.test.js:95-96 — 죽은 분기 문구를 검증하는 테스트(통과하지만 stale).
기대: SH-3 완료 시 잔재 제거. 실제: 3곳 잔존. 재현: backend가 반환하지 않으므로 UI 노출은 재현 불가 — 코드 grep "CRAWL_PENDING"로만 확인됨. 심각도 LOW(비차단·죽은 코드). ※ "죽은 코드/문구 정리"는 code-reviewer 영역과 겹치나, 수용 기준이 명시적으로 "CRAWL_PENDING 잔재 없음"을 요구하므로 QA가 기록한다.

4. 2변형 + 기본 숨김 (신규·레거시) — PASS

frontend가 한때 "신규 설치 기본숨김 미해결"로 표기했던 쟁점을 코드+테스트+경험적 실행으로 해결 확정:

  • 신규 설치 기본 숨김: uiStateStore.defaultState()가 hiddenWidgets: DEFAULT_HIDDEN_WIDGETS.slice()로 시드, DEFAULT_HIDDEN_WIDGETS === SHELF_WIDGET_IDS === ['shelf','shelfWide']. 경험적: defaultState().hiddenWidgets에 shelf·shelfWide 둘 다 포함 확인.
  • 레거시 기본 숨김(이행): normalizeState가 schemaVersion<2 또는 부재(undefined)를 레거시로 분기해 unionShelfHidden 적용. 경험적: v1·undefined 둘 다 union 숨김 확인.
  • 이행 4케이스 테스트 통과: ①v1→union ②undefined→union ③v2 un-hide→재숨김 안함(멱등) ④v2 무변경→멱등. 경험적으로 v2 un-hide 시 재숨김 안 됨 확인(사용자 선택 존중).
  • HOME enum 동형(10섹션): uiStateStore.HOME_SECTION_IDS ↔ app.js L648 동일 10개(attention·productivity·activity·todos·mail·disk·aiusage·shelf·shelfWide·featureAdd), homeLayout-equivalence 테스트가 교차검증.
  • 2변형 렌더: shelf=일반 컬럼, shelfWide=home-section--wide(column-span:all)로 전체폭. 둘 다 renderHomeShelf() 공유 — 위젯 갤러리에서 골라 추가.

5. 열기 동작 분기 — PASS

주입 ctx(mock shell/safeExec/resolveBin) + 실제 temp 폴더·파일로 경험적 실행:

  • file → shell.openPath(real): OS 기본앱. 호출 인자가 저장 실경로와 일치 확인.
  • folder → safeExec(code, [real], {shell:false, detached:true}): VS Code, 인자 [real] 고정·shell:false 확인. 도구 미설정 시 shell.openPath 폴백 경로 존재.
  • url → shell.openExternal(v.value): config.validateHttpUrl로 http/https 재검증 후 열기 (javascript:/file: 등 임의 스킴 차단).
  • 열기 시점 pathPolicy.gate 재-canonicalize + fs.statSync 유형 재확인(TOCTOU 축소), id별 in-flight 상한(MAX_INFLIGHT_OPEN=2).

6. 토글 영속·라이브 반영 — PASS

  • 영속: setSettings가 config.persistConfigKeys({shelfAutoRefresh})로 0600 영속, DEFAULTS.shelfAutoRefresh=true.
  • list 초기값: list().autoRefresh = getAutoRefresh(ctx) — 경험적으로 false 설정 후 list 반영 확인.
  • 라이브 반영(다음 tick): register.js shelfCtx()는 얕은 복사라 .config가 같은 참조. setSettings의 ctx.config.shelfAutoRefresh=val 변이를 scheduler isEnabled:()=>!(ctx.config.shelfAutoRefresh===false)가 동일 객체에서 즉시 읽음. 경험적으로 setSettings(false) 직후 getAutoRefresh·list 모두 false 확인.
  • 스케줄러 게이트: D-SCHED-1(off→tick 즉시 return·egress 0), D-SCHED-2(elevated 스킵), 첫 tick 지연, 동시 크롤은 urlMeta 전역 세마포어 공유 — 코드·테스트(shelf-scheduler 7개) 확인.
  • setSettings BAD_INPUT: boolean 외 입력은 {ok:false,code:'BAD_INPUT'}(경험적 확인).

8. GUI 스모크 — 헤드리스 (코드/테스트 대체)

본 검증 환경은 헤드리스라 npm start 실기 GUI 스모크 불가. Electron 메인 프로세스의 창 생성·디스플레이가 없어 실행해도 무의미. 따라서 위 1~7의 코드 교차대조 + node --test 1090 전수 + 경험적 실행 스크립트(open/toggle/이행/shape)로 동작을 확인했다. 아래는 사용자가 실기에서 확인할 스모크 체크리스트다.
  • 위젯 갤러리에서 '즐겨찾기 셸프'·'즐겨찾기 셸프 (와이드)'가 기본 숨김 상태로 보이고, 추가하면 홈에 나타난다.
  • shelfWide 추가 시 카드가 전체폭으로 스팬된다.
  • 유형=폴더로 실제 폴더 경로 입력 → 스캔 중 표시 후 스파인 추가, 활성 펼침에 디렉토리 메타 표시.
  • 유형=파일로 파일 경로 입력 → 파일 메타(크기·수정시각) 표시.
  • 유형=URL로 사이트 붙여넣기 → 크롤링 중 → og:image 배너(있으면) 또는 색 그라데이션.
  • 스파인 클릭 시 펼쳐지고 활성 카드가 자동으로 적정 위치로 스크롤된다.
  • 셸프 행을 마우스 드래그하면 가로 스크롤, 드래그 직후 클릭으로 잘못 활성화되지 않는다.
  • 휠(세로)로 셸프 가로 스크롤된다.
  • 폴더 항목 열기 → VS Code로 열림(미설정 시 OS 기본앱).
  • 파일 항목 열기 → OS 기본앱으로 열림.
  • URL 항목 열기 → 외부 브라우저로 열림.
  • 항목 삭제 → 즉시 사라지고 토스트 표시.
  • 푸터 자동재크롤 스위치 토글 → 상태/힌트 문구 즉시 변경, 앱 재시작 후에도 유지(영속).
  • 잘못된 경로/주소 입력 → 한국어 에러 박스(시스템/민감 경로는 거부).
  • 잘못된 URL("곧 지원돼요" 문구가 절대 나오면 안 됨 — BUG-SHELF-01 회귀 확인).
