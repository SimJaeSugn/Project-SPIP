Project-SPIP — 사용자 스토리 백로그

PRD(docs/PRD.html)의 요구를 에픽 → 스토리로 분해한다. 각 스토리는 수용 기준·우선순위(MoSCoW)·규모 추정 힌트(S/M/L)·연결 R-ID를 가진다. 추적성 매트릭스의 입력으로 사용한다.

규모 힌트: S=작음(≤1일) · M=보통(2~3일) · L=큼(주 단위/분할 권장)

EPIC A — 스캔 & 탐지

흩어진 프로젝트를 빠짐없이, 그러나 불필요한 폴더는 건너뛰며 찾아낸다.

ID스토리 / 수용 기준우선규모R-ID
US-A1개발자로서, 내 프로젝트 위치를 파악하기 위해 지정 루트를 스캔하고 싶다.
AC: .git/package.json/.vscode 보유 폴더를 1건으로 탐지; 결과를 데이터 파일로 저장; 콘솔에 개수·소요시간 요약.		MustMR-01,14
US-A2개발자로서, 스캔이 빠르도록 거대 폴더를 건너뛰고 싶다.
AC: node_modules·.git 내부·dist/build/.cache 등에 재귀 진입 안 함; 제외 목록 설정 추가 가능.		MustSR-02
US-A3개발자로서, 스캔 대상 루트를 직접 지정하고 싶다.
AC: 설정 파일/CLI 인자로 N개 루트 지정; 미설정 시 기본값 사용 + 안내.		MustSR-04
US-A4개발자로서, 위치를 모르는 프로젝트까지 찾으려 전체 드라이브를 탐색하고 싶다.
AC: --all-drives 옵션으로 모든 드라이브 대상; 제외 규칙 적용; 시작 전 시간 경고.		ShouldMR-05
US-A5개발자로서, 스캔이 멈추지 않도록 순환 링크/깊은 트리를 안전 처리하고 싶다.
AC: 깊이 한도에서 중단·경고; 무한 루프 없음.		ShouldSR-03

EPIC B — 인사이트

각 프로젝트의 상태를 4종 인사이트로 한눈에 이해한다.

ID스토리 / 수용 기준우선규모R-ID
US-B1개발자로서, 무슨 기술로 만든 프로젝트인지 알기 위해 언어/스택을 보고 싶다.
AC: package.json deps·확장자 비율로 주 언어/스택 추정; 전체 분포 집계; 불명 시 "Unknown".		MustMR-06
US-B2개발자로서, 방치된 프로젝트를 가려내기 위해 신선도를 보고 싶다.
AC: 최종 수정일·최근 커밋 시각 표시; 기준일(기본 90일) 초과 시 stale 배지; 기준 설정 가능.		MustMR-07
US-B3개발자로서, 커밋/푸시를 놓친 곳을 찾기 위해 git 상태를 보고 싶다.
AC: 저장소 여부·브랜치·미커밋 변경·ahead/behind 표시; 미설치/비저장소는 "N/A".		MustMR-08,13
US-B4개발자로서, 디스크 정리를 위해 규모/의존성을 보고 싶다.
AC: 총 용량·node_modules 크기·의존성 개수 표시; 무거우면 지연/근사 측정.		ShouldLR-09

EPIC C — 대시보드

목록·통계를 한 화면에서 보고, 원하는 것을 빠르게 찾는다.

ID스토리 / 수용 기준우선규모R-ID
US-C1개발자로서, 전체를 한눈에 보기 위해 카드/리스트 + 통계 요약을 보고 싶다.
AC: 항목별 이름·경로·설명·언어·신선도·git 배지; 상단 통계(개수·분포·stale·총용량); 0건 시 빈 상태 안내.		MustLR-10
US-C2개발자로서, 원하는 프로젝트를 빨리 찾기 위해 필터/정렬/검색하고 싶다.
AC: 언어·신선도·git로 필터(AND); 이름·수정일·용량 정렬; 이름/경로 검색; 클라이언트 측 갱신, 200개 로드 상태에서 입력→재렌더 performance.now() 중앙값 ≤200ms(PRD §3 측정 정의).		MustMR-11
US-C3개발자로서, 최신 상태를 보기 위해 대시보드에서 재스캔하고 싶다.
AC: '재스캔' 클릭 시 서버가 스캔 재실행 후 화면 갱신.		CouldMR-16

EPIC D — 액션 & 견고성

찾은 프로젝트를 바로 열고, 어떤 환경에서도 안전하게 동작한다.

ID스토리 / 수용 기준우선규모R-ID
US-D1개발자로서, 작업을 재개하기 위해 카드에서 바로 VS Code로 열고 싶다.
AC: 클릭 시 code <path> 실행; 화이트리스트 검증; 성공/실패 피드백; code 미설치 시 안내(크래시 없음).		MustMR-12,13
US-D2개발자로서, 도구가 죽지 않도록 git/code 부재·권한 문제를 안전 처리하길 원한다.
AC: 미설치·권한없음·손상파일에도 해당 항목만 스킵/N-A, 전체는 정상 완료.		MustSR-13,N-05
US-D3개발자로서, 진행 상황을 알기 위해 스캔 진행 표시를 보고 싶다.
AC: 1초 이상 소요 시 처리 경로/카운트 주기 표시.		ShouldSR-15

EPIC E — 보안 & 운영 (비기능)

로컬 전용 도구로서 안전과 가벼움을 보장한다.

ID스토리 / 수용 기준우선규모R-ID
US-E1사용자로서, 데이터가 외부로 나가지 않도록 서버가 localhost에만 바인딩되길 원한다.
AC: 127.0.0.1 바인딩; 외부 전송 없음.		MustSN-02
US-E2사용자로서, 임의 파일이 노출되지 않도록 경로 순회가 차단되길 원한다.
AC: 스캔 루트 밖·../ 접근 차단; 열기/읽기는 등록 경로 화이트리스트 내만 허용.		MustMN-03
US-E3사용자로서, 설치가 가볍도록 외부 의존성이 최소이길 원한다.
AC: Node 16+/CommonJS; UI는 빌드/프레임워크 없는 순수 HTML/CSS/JS.		MustSN-04
US-E4사용자로서, 스캔이 빠르게 끝나도록 성능 기준이 충족되길 원한다.
AC(N-01): 200개 프로젝트 규모 지정 루트 스캔이 30초 이내 완료(콜드 캐시 3회 중앙값, 벤치 픽스처). 필터/검색 갱신은 200개 로드 상태에서 입력→재렌더 performance.now() 중앙값 ≤200ms. 측정은 QA 벤치/브라우저 계측으로 자동화.		MustMN-01,R-11
US-E5사용자로서, 내 데이터가 안전·로컬에만 보관되고 인증 부담이 없길 원한다.
AC(N-06): 스캔 결과 JSON은 OS 사용자 캐시 폴더(%LOCALAPPDATA%/~/Library/Caches/$XDG_CACHE_HOME)에 영속화하여 프로젝트 폴더를 오염시키지 않음; 외부 전송 없음; 단일 사용자·인증 없음으로 동작.		MustSN-06
US-E6사용자로서, 키보드와 색약 환경에서도 대시보드를 쓸 수 있길 원한다.
AC(N-07): 주요 인터랙티브 요소(카드·필터·열기 버튼)는 키보드 포커스 이동·Enter 실행 가능; 상태 배지는 색상에만 의존하지 않고 텍스트 라벨 병기; 본문/배지 대비 WCAG AA 권고 충족. MVP 비포함(M4 정식 검증). MVP에서는 권고 수준 적용.		ShouldMN-07
