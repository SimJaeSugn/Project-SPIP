3. API 계약 합의 지점 (S4 · PM 중재)

프론트(public/)와 백엔드(server.js) 경계의 엔드포인트·요청/응답 shape는 S5/S6 병렬 착수 전에 반드시 확정한다(안티패턴: 계약 합의 전 병렬 착수 = 경계면 불일치 주원인). 합의 결과는 docs/architecture/api-contract.html에 자족형 HTML로 기록하고 PM이 합의를 중재한다. 기준은 설계 §9이며, 아래 표가 합의 체크리스트다.

메서드경로요청 shape응답 shape (성공)합의 포인트
GET/—public/index.htmlP1-1 URL 규약
GET/static/<asset>—public/<asset> + MIME + CSP 헤더자산 참조 경로
GET/api/projects—{ schemaVersion, generatedAt, hasSnapshot:boolean, projects: Project[] }Project 필드(§8.1) 합의
GET/api/stats—{ total, byLanguage:{...}, staleCount, totalBytes:null, generatedAt }totalBytes=null(P2-2)
GET/api/health—{ ok:true, hasSnapshot, codeCli:boolean, git:boolean }CLI 가용성 노출
POST/api/open{ id: string } + 세션토큰{ ok:true, code:"OPENING", message }id 기반·M-1 헤더(P2-3)
POST/api/rescan—MVP: 미마운트 → 404 NOT_FOUNDM4 자리만(P3-3)

오류 응답 계약 (합의 필수 · L-3)

  • 200 + ok:false: CODE_CLI_NOT_FOUND | PATH_GONE (R-13 graceful)
  • 403: PATH_NOT_ALLOWED(화이트리스트 밖) · FORBIDDEN_ORIGIN(Host/Origin/토큰, M-1)
  • 404: ID_NOT_FOUND(P2-3) · NOT_FOUND(미존재 자산/rescan@MVP)
  • 스냅샷 부재/손상: 503 아님 → 200 + hasSnapshot:false + projects:[](P2-5 통일, 503 폐기)
  • L-3: 모든 message는 고정 안내문만. 원시 예외·스택·내부 절대경로는 응답 금지(콘솔 로그만).
합의 산출물 DoD: 위 모든 엔드포인트의 요청/응답 shape, Project 필드 타입, 오류 코드 집합, 세션토큰 전달 방식(헤더명/주입 위치), CSP 정책 문자열이 api-contract.html에 명문화되고 frontend·backend·tech-lead·PM가 합의 서명(체크). 이 게이트 통과 전 S5·S6의 경계 코드는 착수 금지.

7. 테스트 전략 (외부 의존성 0 제약)

7.1 러너 선정 — Node 내장 node:test + assert

ADR-001(외부 의존성 0)·기술스택 §10·N-04에 따라 Jest/Mocha/Vitest 등 외부 프레임워크를 도입하지 않는다. Node 16+에 내장된 node:test(test runner)와 node:assert로 단위·통합·벤치를 모두 작성한다.

근거(ADR 정합): ① 외부 의존성 0 유지(ADR-001·N-04) — 별도 설치/번들 없음 · ② Node 16의 node:test는 안정 사용 가능(실험 플래그 가능성은 S0에서 실측, 미흡 시 순수 assert 기반 러너로 폴백 — 그래도 의존성 0 유지) · ③ --test 러너로 회귀·벤치 자동화 가능(§10 테이블 "Node 내장 node:test + assert"). 외부 러너가 꼭 필요하면 별도 ADR로 근거·번들 영향을 남긴다(§10 의존성 추가 정책).

  • og:image 상한 = 100KB (정합 완료). 설계 SSOT(§2·§9·§11)도 100KB로 정정됨 — 코드·테스트·계약·로드맵·설계 5문서 통일. SVG 제외.
  • CSP 무변경. 배너는 data:URI(img-src 'self' data: 기존 허용)·렌더 스타일은 CSSOM(style.cssText — style-src 'self' 무관). 마크업 style 속성/innerHTML 금지(L-1) 준수 확인.
  • egress 신규 표면. SSRF IP 핀(연결 IP=검증 IP)·hop 재검증·압축 미요청. validateHttpUrl 일반화가 기존 briefing 호출을 깨지 않는지 회귀 확인.
  • 경로 표면. add·open 양쪽 canonicalize+pathPolicy 재게이트, 열기 인자 [real] 고정, file=openPath/folder=safeExec(VS Code) 분기(PM #2) 일치.
